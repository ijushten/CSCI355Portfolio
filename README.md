Personal Portfolio Website Project

Project Overview

This project is a personal portfolio website built using HTML, CSS, and JavaScript. The purpose of the website is to present my professional identity, background, skills, projects, and contact information. The portfolio is designed to be simple, clean, responsive, and easy for visitors to navigate.

The website includes sections for Home, About, Skills, Projects, Schedule a Meeting, and Contact. It also includes interactive features such as a mobile navigation menu, a contact form response message, and a meeting request form.

Part 2: Design Planning

1. What overall style will best represent you?

A clean, simple, and professional style best represents me. The design should look organized and modern without being too complicated.

2. What color scheme will you use and why?

The color scheme uses blue, white, light gray, and dark text. Blue is used as the main accent color because it gives the site a professional and technology-focused look.

3. What fonts will you use for headings and body text?

The website uses Arial, Helvetica, and sans-serif fonts.

4. How will your design reflect your personality or field?

The design reflects my field by being clear, structured, and practical. the layout uses clean sections, cards, and simple navigation.

5. What layout will your homepage follow?

The homepage begins with a hero section that includes my name, role, short introduction, action buttons, and a profile image. This gives visitors the most important information first.

6. How will you organize project sections visually?

Projects will be organized as cards in a grid layout. Each card includes a project title, short description, and the tools or technologies used.

7. Will the site be mobile-friendly? How will you ensure responsiveness?

Yes. The site will be mobile-friendly. Responsiveness is handled with CSS media queries. On smaller screens, the layout changes from multiple columns to one column, and the navigation becomes a mobile menu.

8. What visual hierarchy will guide visitors?

The visual hierarchy starts with the largest text, which is my name in the hero section. Section headings guide users through the page, and cards separate skills and projects. Buttons and links use the accent color so visitors know where they can click.

9. How will consistency be maintained across pages?

Consistency is maintained by using the same colors, spacing, fonts, card styles, button styles, and section widths throughout the entire site.

10. How will accessibility be considered?

Accessibility is considered by using readable font sizes clear labels for form fields, alt text for the profile image, and a responsive layout that works on different screen sizes.

11. Will you use icons, images, or illustrations? Why?

I will include a profile image so the website feels more personal. I am keeping the visuals simple so the focus stays on my content, skills, and projects.

Part 3: Interactivity Planning

1. What interactive elements will your site include?

The site includes these interactive elements:

Navigation links
Mobile menu button
Resume button
Project button
Contact form
Meeting request form
email link
GitHub link

2. Will your site include a contact form? How will it work?

Yes. The contact form includes fields for name, email, and message. When a visitor submits the form, JavaScript prevents the page from refreshing and displays a confirmation message. 

3. What JavaScript features will you implement?

The JavaScript features include:

Toggling the mobile navigation menu
Handling the contact form submission
Displaying a contact form message
Handling the meeting request form submission
Checking that the meeting form includes a name, date, and time
Displaying a meeting confirmation message

4. How will users receive feedback from interactions?

Users receive feedback through text messages on the page. For example, when the contact form is submitted, a message appears telling the user their message has been prepared. When the meeting form is submitted, the page confirms the selected meeting date and time.

5. How does interactivity improve the user experience?

Interactivity improves the user experience because it makes the website feel responsive and easier to use. The mobile menu helps visitors navigate on small screens, and the form messages let users know that their actions were received.
